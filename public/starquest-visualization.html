The Quest for Stars

StarQuest turns everyday tasks into epic family adventures! Parents create quests, children earn shining stars, and everyone grows together. Built with Next.js 15 + Supabase — bilingual, secure, and fun.

ACT 1 — WHAT

Once upon a time, in a family far away...

Parents wished they could turn chores and good habits into a game. StarQuest was born — a magical star chart where kids complete quests, earn stars, and trade them for awesome rewards!

StarQuest classifies every quest along two dimensions: Type (bonus, duty, violation) and Scope (self, family, others). This creates a rich 3x3 matrix of behaviors. Children only see bonus quests — keeping their experience positive. Parents see all types to track the full picture. The app speaks both English and Chinese.
ACT 2 — WHO

Meet the heroes behind the curtain

Six characters work together: the Frontend (storefront window), the Server (the brain), the Database (a vault that never forgets), Auth (the security guard), Email (the messenger owl), and Analytics (the wise oracle).

🌐 Frontend (React + Tailwind) — Colorful LEGO bricks for every screen. Uses next-intl for bilingual magic. 🧠 Server (Next.js 15) — Pre-cooks pages for speed. Runs API routes and cron jobs. 🗄️ Database (Supabase + PostgreSQL) — Stores families, quests, stars, rewards. Row-Level Security: each family only sees their own data. 🔒 Auth (Middleware) — Checks every visitor's badge before letting them in. 📧 Email (Resend) — Sends weekly reports, monthly summaries, and invitation emails. 📊 Analytics (PostHog) — Privacy-first tracking. Children are anonymous!
ACT 3 — CREATE

The adventure begins: Parent creates a quest

Mom opens StarQuest and creates 'Help with Dishes' as a Bonus quest worth 15 stars. She picks Family scope and Chores category. The quest is saved — ready for Alisa to discover!

Quest classification: Bonus (earn stars), Duty (lose stars if missed), Violation (lose stars when broken). Scope: Self, Family, Others. Parents can set a star multiplier (1x-10x) for exceptional effort!
ACT 4 — DISCOVER

Alisa discovers the quest

Alisa opens her phone and sees 'Help with Dishes' bouncing in. She only sees bonus quests — duty and violation quests are hidden, keeping her experience positive and fun!

The child view uses getChildVisibleQuests() to filter. This is a core design principle: children see encouraging bonus adventures, parents see the full picture. Filtering runs server-side — filtered data never reaches the child's browser.
ACT 5 — EARN

Stars requested, stars approved!

Alisa washes all the dishes and taps 'Request Stars' with a note. The request goes pending. Mom reviews, clicks Approve, and Alisa's balance jumps from 265 to 280 stars!

Flow: INSERT star_transactions (pending) -> parent reviews -> UPDATE (approved) -> child_balances VIEW auto-recalculates. Parents can batch-approve or set custom dates for backdating.
ACT 6 — COLLABORATE

How the characters work together

服务端："让我问问认证。"-> 认证："没问题！"-> 数据库："已保存！待审批。"-> 邮件："妈妈，新请求！"-> 分析："已记录。"">Frontend: 'Alisa wants stars!' -> Server: 'Let me check with Auth.' -> Auth: 'She's legit!' -> Database: 'Saved! Pending.' -> Email: 'Hey Mom, new request!' -> Analytics: 'Tracked it.'

Maps to real code: React fetch('/api/...') -> Next.js API route -> middleware.ts checks session -> Supabase INSERT -> PostHog captures event -> Cron job emails weekly summary. All tables use family_id + RLS — PostgreSQL itself refuses cross-family queries.
ACT 7 — SUPERPOWERS

Secret superpowers of StarQuest

💳 Credit System — borrow stars with real interest! 📧 Auto-Reports — weekly/monthly emails. 🎭 Demo Mode — one-click, 30 days of data. 🧪 2,849 tests, ~99% coverage!

Credit: credit_settings, credit_transactions, interest_tiers. Kids borrow now, repay with interest — teaching financial literacy! Reports: Cron at /api/cron/daily-jobs generates Markdown via fetchReportBaseData() + buildChildrenStats(), sends via Resend. Demo: POST /api/seed-demo creates family with 45 quests, 11 rewards, 2 children, 30 days of activity. Passwordless magic link login. Testing: Jest + React Testing Library. Every component, hook, API route tested.

Security Features

🛡️

Row-Level Security

Every table uses family_id with PostgreSQL RLS. Each family can only access their own data — enforced at the database level.

🔒

Middleware Auth

Intl middleware runs first, then Supabase session check. This order is critical — reversing it breaks sessions.

👥

Role-Based Access

Parents route to /admin, children to /app. requireParent() and requireAuth() guards protect every route.

🔐

Privacy-First Analytics

Children identified by UUID only — no email, no name. Session recordings disabled for all child accounts.

⚡

SECURITY DEFINER

Sensitive operations (password reset, email update, child delete) run via SECURITY DEFINER functions — never exposed to client.

🍪

Cookie-Based Sessions

httpOnly cookies via @supabase/ssr. Post-auth uses window.location.href (not router.push) to avoid login loops.

The Database Memory 🗄️

Everything in StarQuest is saved in a PostgreSQL database. Tables hold the data, and arrows show how they connect!

🔒 Row-Level Security (RLS): Every table has a magic shield — you can only read or write data that belongs to your own family. Even if someone guessed another family's ID, the database refuses! This is built right into PostgreSQL.
